CC-Switch 使用

CC Switch 是一款强大的跨平台 API 供应商切换与管理工具，最初为 Claude Code 设计，现已扩展支持 Codex、Gemini CLI、OpenCode 和 OpenClaw 等多个开发工具前端。

下载软件

前往 Github Release 页面下载最新的安装包：https://github.com/farion1231/cc-switch/releases。请根据你当前使用的系统选择对应版本的安装包；Windows 一般建议选择 .msi 安装包，macOS 和 Linux 则选择各自对应的平台版本即可。

Codex配置步骤

前置说明：使用 CC-Switch 配置 Codex 的前提是你已经安装好 CodeX CLI；如果尚未安装，请先前往 CodeX配置 / 安装教程 完成安装后再继续下面的步骤。

步骤 1：添加供应商

打开软件后，选择codex选项并点击右上角的 “+” 加号图标，新建一个 API 供应商配置。

步骤 2：填写供应商配置信息

  • 供应商名称： codexsurf （可自定义）
  • 官网链接： https://codesurf.ccwu.cc/ （选填）
  • API Key： 填入你在控制台获取的令牌
  • API 请求地址： https://codesurf.ccwu.cc/
  • 模型名称： 例如 gpt-5.4

步骤 3：修改供应商测试模型

由于 CC 内部测试模型默认为 gpt-5.1-codex（或更低版本），属于旧版本模型，而 codexsurf 平台已移除旧版本模型以保证用户体验，所以这里需要将测试模型改为平台当前支持的模型，例如 gpt-5.4。

如果不做这一步修改，测试时可能会出现 503 或 401 报错，导致测试失败，但这通常不影响正常使用；如非必要，也可以忽略此步骤。

上述步骤完成后保存编辑即可正常使用。

注意事项

  • 请确保填写的请求地址结尾**不要带多余的斜杠 / 或 **/v1，CC Switch 和底层命令行工具会自动处理路径拼接。
  • 每次在 CC Switch 中点击切换供应商后，你需要重启当前正在运行的命令行终端会话，让新的配置生效。
