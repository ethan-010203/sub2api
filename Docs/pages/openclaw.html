OpenClaw 配置

OpenClaw 是一个开源（MIT 许可）的自托管网关，可将 WhatsApp、Telegram、Discord、iMessage 等聊天应用连接到 AI 编码代理，支持多代理路由、工具使用和会话记忆。更多信息请参考 OpenClaw 官网。本教程将指导你如何在 OpenClaw 中安装并配置 CodexSurf API。

安装教程

OpenClaw 版本更迭较快，本文不再单独维护具体安装命令，以免和当前版本不一致。请直接参考 OpenClaw 官方安装教程文档，按你当前系统和最新版本说明完成安装。

使用教程

配置 OpenClaw

配置文件位置

macOS / Linux：

path 
~/.openclaw/openclaw.json

Windows：

path 
%USERPROFILE%\.openclaw\openclaw.json

注意：如果文件不存在，可以手动创建。首次运行 openclaw doctor 也会自动生成默认配置。

CodexSurf 配置示例

获取令牌：请先前往 获取 API Key 页面创建并复制 codexsurf 令牌，再填入下方配置。

由于 OpenClaw 配置较为复杂，这里仅展示最小可用配置。其他配置可参考 OpenClaw 官方文档，或前往各类媒体平台查找相关攻略。

json 
{
    "gateway": {
        "mode": "local",
        "auth": {
            "mode": "token",
            "token": "此处填写你的gateway token"
        }
    },
    "agents": {
        "defaults": {
            "workspace": "~/.openclaw/workspace",
            "model": {
                "primary": "codexsurf/gpt-5.4"
            }
        },
        "list": [
            {
                "id": "main"
            }
        ]
    },
    "models": {
        "mode": "merge",
        "providers": {
            "codexsurf": {
                "baseUrl": "https://codesurf.ccwu.cc/v1",
                "apiKey": "此处粘贴您的codexsurf令牌",
                "api": "openai-responses",
                "headers": {
                    "User-Agent": "Mozilla/5.0"
                },
                "models": [
                    {
                        "id": "gpt-5.4",
                        "name": "gpt-5.4",
                        "reasoning": false,
                        "input": [
                            "text"
                        ],
                        "cost": {}
                    }
                ]
            }
        }
    }
}

启动并验证

配置完成后，使用下面的命令启动 gateway：

bash 
openclaw gateway

如果 gateway 已经在运行，则使用下面的命令重启：

bash 
openclaw gateway restart

另开一个终端，使用下面的命令测试是否能够正常回复：

bash 
openclaw agent --agent main --message "hi"

如果能正常收到回复，说明配置成功。

OpenClaw 其余配置请直接参考 OpenClaw 官方文档。

常见问题

问题 解答
遇到 401 报错怎么办？ 优先检查 apiKey 是否填写正确，确认当前配置中粘贴的是可用的 codexsurf 令牌。
遇到 404 报错怎么办？ 优先检查 "baseUrl": "https://codesurf.ccwu.cc/v1" 是否填写正确，尤其注意不要漏掉末尾的 /v1。这一点和 CodeX 配置不同，OpenClaw 这里需要带上 /v1。
遇到 403 报错怎么办？ 优先检查是否已经添加 "headers": { "User-Agent": "Mozilla/5.0" }。由于 OpenClaw 使用的协议不同，这里通常需要显式补上这个请求头。
遇到 503 报错怎么办？ 优先考虑模型版本问题。你可以在自己的订阅页面查看当前订阅支持哪些模型；如果在配置中填入了当前订阅不支持的模型，也可能会触发 503 报错。
