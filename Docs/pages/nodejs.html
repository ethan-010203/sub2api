Node.js 前置安装

CodeX CLI 和 OpenClaw 都依赖 Node.js 运行环境。开始配置这些工具前，建议先完成本页的安装与验证步骤。

什么时候需要先安装 Node.js？

  • CodeX 配置： CodeX 通过 npm 安装，因此需要本机具备可用的 Node.js 环境。
  • OpenClaw 配置： 如果你通过 npm 或 pnpm 安装 OpenClaw，通常建议使用较新的 Node.js 版本。

Windows 安装

方式一：使用官网安装包（推荐）

  1. 访问 https://nodejs.org/。
  2. 下载 LTS 版本的 Windows Installer（`.msi`）。
  3. 运行安装程序，并按默认设置完成安装。
  4. 安装完成后，重新打开终端窗口。

方式二：使用包管理器

使用 Winget：

powershell 
winget install OpenJS.NodeJS.LTS

使用 Chocolatey：

powershell 
choco install nodejs-lts

使用 Scoop：

powershell 
scoop install nodejs-lts

macOS 安装

方式一：使用 Homebrew（推荐）

如果尚未安装 Homebrew，可以先执行：

bash 
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

然后安装 Node.js：

bash 
brew install node

方式二：使用官网安装包

  1. 访问 https://nodejs.org/。
  2. 下载 LTS 版本的 macOS Installer（`.pkg`）。
  3. 运行安装程序，并按默认设置完成安装。

Linux 安装

Ubuntu / Debian

bash 
sudo apt update
curl -fsSL https://deb.nodesource.com/setup_lts.x | sudo -E bash -
sudo apt-get install -y nodejs

CentOS / RHEL / Fedora

bash 
sudo dnf install nodejs npm
# 或 sudo yum install nodejs npm

Arch Linux

bash 
sudo pacman -S nodejs npm

验证安装

安装完成后，在终端中执行以下命令确认版本信息已经正常输出：

bash 
node --version
npm --version

版本建议

如果你主要使用 CodeX，优先选择 Node.js 的 LTS 版本即可；如果你准备安装较新的 OpenClaw 版本，也建议尽量保持 Node.js 版本较新，避免依赖兼容问题。
