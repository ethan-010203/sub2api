CodeX配置

CodeX 是 OpenAI 推出的命令行编程助手，可用于代码生成、修改、审查与日常开发协作。

安装教程

前置说明：CodeX CLI 依赖 Node.js 环境，请先完成 Node.js 前置安装，再继续下面的安装和使用步骤。

安装 CodeX CLI

bash 
npm i -g @openai/codex
codex --version

升级 CodeX CLI

如果已经安装过旧版本，可以执行下面的命令升级到最新版本：

bash 
npm i -g @openai/codex@latest
codex --version

使用教程

方式一：手动配置

Windows 平台

1. 创建配置目录

powershell 
mkdir %USERPROFILE%\.codex
cd %USERPROFILE%\.codex

2. 创建配置文件

在 %USERPROFILE%\.codex\ 目录下创建 config.toml 文件，内容如下：

toml 
model_provider = "codexsurf"
model = "gpt-5.4"
review_model = "gpt-5.4"
model_reasoning_effort = "xhigh"
disable_response_storage = true
network_access = "enabled"
windows_wsl_setup_acknowledged = true
model_context_window = 1000000
model_auto_compact_token_limit = 900000

[model_providers.codexsurf]
name = "codexsurf"
base_url = "https://codesurf.ccwu.cc"
wire_api = "responses"
requires_openai_auth = true

3. 创建认证文件

在同一目录下创建 auth.json 文件：

获取令牌：请先前往 获取 API Key 页面创建并复制 CodeX 专用令牌，再填入下方配置。

json 
{
  "OPENAI_API_KEY": "此处粘贴您的codexsurf令牌"
}

常见问题

问题 解答
CodeX 和 Claude Code 的令牌通用吗？ 不通用。Claude Code 使用 Claude 令牌组，CodeX 使用 CodeX 专用令牌组，请在 CodexSurf 控制台选择对应的令牌分组。
配置文件放在哪里？ Windows：%USERPROFILE%\.codex\；macOS / Linux：~/.codex/
API 请求失败怎么办？ 检查 API Key 是否正确，Base URL 是否为 https://codesurf.ccwu.cc，以及网络是否可以正常访问 CodexSurf
如何切换模型？ 修改 config.toml 中的 model 字段即可
遇到 stream disconnected before completion: stream closed before response.completed 怎么办？ 优先检查当前 Node.js 和 Codex 是否已经升级到最新版本。如果版本不是最新，先升级版本后再重试；这类流式中断问题很多时候和旧版本兼容性有关。
遇到 503 报错怎么办？ 优先检查当前配置里填写的模型版本是否受你的订阅支持。你可以在自己的订阅页面查看可用模型；如果在配置中填入了当前订阅不支持的模型，可能会触发 503 报错。
